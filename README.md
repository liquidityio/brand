# @liquidityio/brand

Liquidity's org config, theme seeds, fonts, and logo assets. Thin wrapper over [`@hanzo/theming`](https://github.com/hanzoai/theming).

## Install

```bash
pnpm add @liquidityio/brand
```

## Usage

### Hanzogui (Tamagui fork)

```tsx
import { hanzoguiConfig } from '@liquidityio/brand'
import { HanzoguiProvider } from 'hanzogui'

<HanzoguiProvider config={hanzoguiConfig}>
  <App />
</HanzoguiProvider>
```

### Tailwind / shadcn

One import in your app's main CSS:

```css
@import '@liquidityio/brand/tw-liquidity.css';
```

Pulls in `tailwindcss`, Liquidity's easing/fonts, the generated palettes, and the shadcn semantic color mappings — in the right order.

### Brand data

```typescript
import { config, brandIdentity } from '@liquidityio/brand'
import type { BrandIdentity, OrgConfig } from '@liquidityio/brand'
import orgConfig from '@liquidityio/brand/liquidity-org-config.json'
import themes from '@liquidityio/brand/themes.json'

// Logo assets
// @liquidityio/brand/assets/logo/logo.svg
// @liquidityio/brand/assets/logo/favicon.svg
// @liquidityio/brand/assets/logo/wordmark.svg
```

## Structure

```
src/
├── liquidity-org-config.json   # identity, domains, chains, rpc, api, iam, walletConnect
├── themes.json                 # 7 theme seeds (used to generate palettes)
├── brand-palettes.css          # generated by prebuild (gitignored)
├── liquidity-tw-additions.css  # org-specific non-color TW tokens (easing, fonts)
├── tw-liquidity.css            # bundle: tailwind + additions + palettes + semantic
├── fonts.ts                    # Liquidity font defs (pplxSans)
├── hanzogui-config.ts          # calls createHanzoguiConfig
├── types.ts                    # BrandIdentity, OrgConfig
└── index.ts                    # re-exports config, brandIdentity, hanzoguiConfig
assets/
└── logo/                       # logo, logo-mono, logo-white, wordmark, favicon (SVG)
```

## Exports

| Path | What |
|------|------|
| `@liquidityio/brand` | `config`, `brandIdentity`, `hanzoguiConfig`, types |
| `@liquidityio/brand/liquidity-org-config.json` | Raw identity + runtime config |
| `@liquidityio/brand/themes.json` | Theme seeds |
| `@liquidityio/brand/hanzogui-config` | Hanzogui config re-export |
| `@liquidityio/brand/tw-liquidity.css` | Single CSS bundle for apps |
| `@liquidityio/brand/assets/*` | Logo files |

## How it works

Almost everything — theme engine, palette generation, components, shadcn mappings — lives in `@hanzo/theming`. This package just provides Liquidity-specific data:

- **`liquidity-org-config.json`** — identity (name, domains, social, etc.), chains, RPC endpoints, API endpoints, IAM, WalletConnect
- **`themes.json`** — 7 theme seeds
- **`fonts.ts`** — pplxSans font definitions
- **`liquidity-tw-additions.css`** — easing curves and font families
- **`hanzogui-config.ts`** — calls `createHanzoguiConfig` from `@hanzo/theming/hanzogui-config` with our seeds and fonts

The `prebuild` script runs `generate-palettes` (shipped by `@hanzo/theming`), which reads `src/themes.json` and writes `src/brand-palettes.css`.

## Build

```bash
pnpm build    # runs prebuild (generate-palettes), then tsup
pnpm tc       # typecheck
pnpm dev      # watch mode
```

## Publish

```bash
pnpm pub               # bump patch
pnpm pub minor         # bump minor
pnpm pub major         # bump major
pnpm pub 2.0.0         # exact version
```

The script builds, bumps version, commits as `v{version}`, tags, and publishes. It refuses to run on a dirty tree. Push is left to you:

```bash
git push && git push --tags
```
